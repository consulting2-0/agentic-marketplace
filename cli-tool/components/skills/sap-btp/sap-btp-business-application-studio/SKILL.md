# SAP Business Application Studio (BAS) Skill

## Dev Space Types

| Dev Space | Included Extensions | Use For |
|---|---|---|
| **Full Stack Cloud Application** | CAP Tools, Fiori Tools, CF Tools, HANA | CAP + Fiori full-stack development |
| **SAP Fiori** | Fiori Tools, UI5 Tools, CF Tools | Fiori Elements & freestyle UI5 |
| **SAP HANA Native** | HANA Tools, HDI, SQLScript | Native HANA development |
| **ABAP** | ABAP Extension Pack, ABAP Cleaner | BTP ABAP Environment development |
| **Basic** | Git, Terminal, Code editor only | Custom/lightweight work |

## Dev Space Configuration

### Essential Extensions to Add
```
For Full Stack:
  ✓ SAP CAP Tools (built-in)
  ✓ SAP Fiori Tools (built-in)
  ✓ SAP HANA Database Explorer (add-on)
  ✓ SAP Cloud Foundry Tools (built-in)
  ✓ ESLint (built-in)

Optional:
  + Kubernetes Tools (for Kyma dev)
  + REST Client (API testing)
  + GitLens (enhanced Git)
  + Thunder Client (API testing)
```

## Connecting to Systems

### BTP Destinations in BAS
```
BAS → hamburger menu → Preferences → Open Settings → BAS Destinations

Required destination properties for Fiori dev:
  WebIDEEnabled: true
  WebIDESystem: S4H_DEV
  WebIDEUsage: odata_gen,dev_abap (for ABAP)
               odata_gen (for OData only)

Add to CF destination:
  HTML5.DynamicDestination: true
  HTML5.Timeout: 60000
```

### CF Login from BAS Terminal
```bash
# Login via SSO
cf login -a https://api.cf.eu20.hana.ondemand.com --sso

# Or with user/password
cf login -a https://api.cf.eu20.hana.ondemand.com \
  -u user@company.com \
  -o my-org \
  -s dev
```

## CAP Development in BAS

### Run Configuration (BAS)
```json
// .vscode/launch.json (auto-generated by BAS)
{
  "configurations": [
    {
      "name": "cds watch (development)",
      "type": "node",
      "request": "launch",
      "cwd": "${workspaceFolder}",
      "runtimeExecutable": "npx",
      "runtimeArgs": ["cds", "watch"],
      "env": {
        "CDS_ENV": "development"
      }
    },
    {
      "name": "cds watch (HANA)",
      "type": "node",
      "request": "launch",
      "runtimeExecutable": "npx",
      "runtimeArgs": ["cds", "watch", "--profile", "hybrid"]
    }
  ]
}
```

### Hybrid Testing (CAP + HANA Cloud from BAS)
```bash
# Bind to real HANA service from BAS terminal
cds bind --to my-hana-hdi      # reads CF service key
cds bind --to my-xsuaa

# Run in hybrid mode (local code + cloud services)
cds watch --profile hybrid
```

## Fiori Development in BAS

### Generate New Fiori App
```
1. Open Command Palette (Ctrl+Shift+P)
2. Run: "Fiori: Open Application Generator"
3. Select:
   - Floorplan: List Report Object Page
   - Data Source: Connect to a system → Choose BTP destination
   - Service: /sap/opu/odata4/...
   - Entity: Select main entity
   - App ID: com.company.myapp
4. Click "Finish" → App scaffolded automatically
```

### Preview with Live Data
```bash
# Start preview with BTP destination
fiori run --open "index.html" --config ./ui5-local.yaml

# Where ui5-local.yaml points to BTP destination:
server:
  customMiddleware:
    - name: fiori-tools-proxy
      configuration:
        backend:
          - path: /sap/opu/odata
            url: https://my-system.company.com
            destination: S4H_DEV
```

## ABAP Dev Space

### Connect to BTP ABAP Environment
```
1. Create ABAP service instance in BTP
2. Create service key
3. In BAS: ABAP view → New ABAP Project → Select service key
4. Develop ABAP artifacts: classes, RAP BOs, CDS views
5. Push directly to BTP ABAP system (ADT integration)
```
